# Ideapick — спецификация

**Продукт:** рекомендательный сервис — от идеи до обоснованной оценки «стоит ли браться» для разработчиков и соло-предпринимателей.  
**Домен:** ideapick.ru · **Язык UI:** русский · **Принцип:** минимум ввода от пользователя, максимум выводов от системы.

```
Идея (своя / каталог / AI по анамнезу) → анализ (AI)
```

---

## Маршруты

| Путь | Кто | Суть |
|------|-----|------|
| `/` | все | Лендинг, цены, CTA «Начать», **+100 ₽ при регистрации** |
| `/login` | гость | Google или magic link → `/app/ideas` |
| `/app/ideas` | auth | Список (grid), фильтр/сортировка, баланс; добавить идею, рандом, анамнез — в UI |
| `/app/ideas/[id]` | auth | Страница идеи: просмотр, анализ / обновление анализа, архив, удаление |
| `/privacy`, `/terms` | все | Политики; оплата — **самозанятость** (НПД), чеки через ЮKassa |

`/app/*` без сессии → `/login?callbackUrl=...`

---

## Идея

**Ввод:** название (≤100), описание + placeholder-подсказка. Аудиторию, нишу, монетизацию не спрашиваем.

**Неизменяемость:** после создания название и описание **не редактируются**. Опечатка или новая формулировка — новая идея; старую можно убрать (см. ниже).

**Жизненный цикл:**

| Состояние | Суть |
|-----------|------|
| **Активная** | В основном списке на `/app/ideas` |
| **Архив** | Скрыта из основного списка; отчёт анализа и история операций сохраняются; можно вернуть в активные |

**Архив vs удаление:**

- **Перенести в архив** — без подтверждения. Можно **вернуть** из архива в активные.
- **Удалить навсегда** — из списка и из страницы идеи (активная или архивная). Без анализа — сразу, без confirm. С анализом — явное предупреждение: текст идеи и отчёт удаляются безвозвратно.

**Список по умолчанию:** только **активные** идеи; архивные не показываются. Фильтр «Архив» — в существующем меню фильтров (рядом с «все / с анализом / без анализа»).

**Теги, вердикт, ручной статус** — вне текущего scope. Оценка и выводы — только из отчёта анализа (см. отдельный документ по механике анализа, когда будет готов).

## Генерация

| Режим | Цена | Механика |
|-------|------|----------|
| **Случайная** | 0 ₽ | Выдача из каталога, без AI. **3 раза в сутки** на пользователя. UI: «из каталога», не «нейросеть». |
| **По анамнезу** | 49 ₽ | Краткий опрос → AI → несколько карточек-идей. |

**Каталог:** выданная идея этому пользователю не повторяется. Исчерпан пул или дневной лимит → кнопка недоступна. Пополнение каталога — вручную бэкендом, UI админки нет.

## Анализ (AI)

**99 ₽** · вход: название + описание (снимок на момент запуска) · ответ — **структурированный JSON** (не простыня).

Детали критериев, секций отчёта и промпта — в [`docs/analysis.md`](./analysis.md).

**Первый запуск:** кнопка «Запустить анализ · 99 ₽» — когда отчёта ещё нет.

**Обновление анализа:** когда отчёт уже есть — «Обновить анализ · 99 ₽» (не «новый анализ»). Тот же текст идеи, тот же пайплайн и цена; AI переоценивает с учётом актуального контекста (рынок, риски и т.п.). На карточке показывается дата последнего анализа. Предыдущий отчёт **полностью заменяется** после подтверждения. История версий не хранится.

Дисклеймер: оценка AI, не совет и не гарантия.

**Надёжность:** ретраи; hold на балансе → списание или возврат (см. монетизацию). UI: состояние загрузки; happy path + нехватка баланса (переход к пополнению).

Нереализованные фичи и продуктовые гипотезы (upsell после анализа, шаблоны идей, история версий, слайды) — в [`docs/backlog.md`](./backlog.md).

## Монетизация

Баланс ₽, без подписки. **ЮKassa**. Мин. пополнение **100 ₽** (пресеты 100/300/500/1000). Приём: **самозанятость** (при росте — ИП).

| Событие | Сумма |
|---------|--------|
| Первая регистрация | **+100 ₽** |
| Анализ / пересчёт | 99 ₽ |
| Генерация по анамнезу | 49 ₽ |
| Рандом, CRUD | 0 ₽ |

**Hold:** сумма блокируется → AI → успех: списание · провал после ретраев: возврат. Доступный баланс = общий − активные hold.

Перед платной операцией: цена + подтверждение. История операций (пополнения и списания) — в интерфейсе пополнения баланса. Return после ЮKassa — на страницу, с которой инициировано пополнение.

## Авторизация

**Better Auth** · без паролей · **без GitHub** (нет надёжно подтверждённой почты без отдельной верификации).

Google — основной; magic link — альтернатива. Один аккаунт на email (Google + почта). Первый вход = регистрация + **100 ₽**.

---

## Нефункциональное

Rate limit (auth, AI, рандом 3/сутки). HTTPS. Удаление аккаунта — каскадно. Деплой: VPS, Docker, Next standalone.
